# Commit Message Prefixes

When creating commit messages, always add one of the prefixes below. This helps contributors understand changes at a glance and enables AI tools to scan and categorize commits easily.

**Format:** `prefix: description`

**Example:** `feat: add user authentication`

Reference: [Qiita - コミットメッセージのプレフィックス](https://qiita.com/muranakar/items/20a7927ffa63a5ca226a)

## Prefix List

| Prefix | Meaning |
|--------|---------|
| `fix` | Fix a bug in existing functionality |
| `hotfix` | Emergency fix requiring urgent attention |
| `add` | Add new files or features |
| `feat` | Add new functionality |
| `update` | Modify existing features (no reported issues) |
| `change` | Changes due to specification updates |
| `clean` | Code cleanup |
| `refactor` | Code refactoring |
| `improve` | Code enhancement |
| `disable` | Temporarily deactivate features |
| `remove` | Remove files or features |
| `delete` | Delete files or features |
| `rename` | Rename files |
| `move` | Relocate files |
| `upgrade` | Version upgrades |
| `revert` | Revert to previous commits |
| `docs` | Documentation modifications |
| `style` | Code formatting and styling |
| `perf` | Performance optimization |
| `test` | Test code modifications or additions |
| `chore` | Build tool updates and miscellaneous changes |
